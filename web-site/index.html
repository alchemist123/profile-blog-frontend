How It Works

A transformer pipeline across
every device in your cluster

PodX splits a model's layers across nodes. Each node gets a contiguous shard, passes activations forward over gRPC, and returns the final logits to the coordinator for token sampling.

Layer Pipeline — Request Flow
1

Install & Start

Run podx start on each machine. The node detects hardware, starts gRPC and HTTP servers, and advertises itself on mDNS.

2

Auto-Discovery

Nodes find each other in seconds via mDNS — the same protocol your printer uses. No IPs to configure. Works across macOS, Linux, and Windows.

3

Load a Model

Run podx model load with any GGUF file. PodX computes the shard map and distributes layer ranges to each node automatically.

4

Inference Ready

Hit /v1/chat/completions — OpenAI-compatible, streaming SSE, authenticated. Your existing code works unchanged.

Architecture

Built in Rust.
Seven focused crates.

PodX is a clean workspace with strict dependency rules. Every crate has one responsibility. No circular dependencies. No bloat.

⚙️ Core crates

Every crate has exactly one job. podx-core is pure types — no logic, no deps. State, inference, and network are independent leaf crates that never import each other.

podx-core: types podx-state: gossip podx-inference: llama podx-network: gRPC podx-api: HTTP podx-cli: commands

📡 gRPC Protocol

Nodes communicate over cluster.proto — a tonic/prost-generated service with four RPCs: Forward, Gossip, Heartbeat, and InstallShard.

Forward — pass activation tensors between layers
Gossip — replicate ClusterState to all peers
Heartbeat — liveness probe between coordinator & workers
InstallShard — push model shards to peer nodes

🗳️ Coordinator Election

Every node scores its own hardware and broadcasts it. The highest score wins. Election is deterministic, tie-broken by node ID. Re-election is instant if the coordinator disappears.

score = RAM×100 + cores×50 + VRAM×200
macOS bonus: +2000 pts
Example M2 16GB Metal: 4300 pts
Example Linux 32GB CUDA: 5100 pts

Drop-in replacement

One line change.
Your code stays the same.

PodX speaks OpenAI's protocol. Change the base URL and you're running on your own hardware.

Before — Cloud API 
from openai import OpenAI

client = OpenAI(
    # Sending prompts to the cloud
    api_key="sk-proj-...",
)

response = client.chat.completions.create(
    model="gpt-4o",
    messages=[
        {"role": "user",
         "content": "Hello!"}
    ],
    stream=True
)

for chunk in response:
    print(chunk.choices[0].delta.content)
After — PodX (local cluster) 
from openai import OpenAI

client = OpenAI(
    base_url="http://localhost:8080/v1",
    api_key="sk-podx-your-key",
    api_key="sk-proj-...",
)

response = client.chat.completions.create(
    model="podx-default",
    messages=[
        {"role": "user",
         "content": "Hello!"}
    ],
    stream=True
)

for chunk in response:
    print(chunk.choices[0].delta.content)
curl — direct API call with streaming 
curl http://localhost:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer sk-podx-your-key" \
  -d '{
    "model": "podx-default",
    "messages": [{"role": "user", "content": "Explain distributed inference in one paragraph."}],
    "stream": true,
    "max_tokens": 512
  }'
